Programmation des interfaces

Les commentaires

Utilisez les commentaires pour vous aider dans l’écriture. Ils vous permettent d’indiquer à quoi serve les différentes lignes.

Ils sont destinés au lecteur du programme, et par conséquent, doivent être compréhensible. Si votre programme n’est destiné qu’à une utilisation privée, mettez de quoi vous rappeler de ce que vous avez écrit.

Il n’existe que des commentaires sur une seule ligne, définis de la manière suivante

; Ceci est un commentaire

Structure des programmes

Les programmes sont structurés en sections. Il existe quatre types de sections : les sections

$
,
$$
(dites double
$
),
*
et
**
. Une fois la section définie, il y a un paramètres et sa valeur par ligne.

Voici leur fonctions.

La création de fenêtre : $$

Le double

$
(
$$
) permet de définir la fenêtre. Il est impérativement suivit de
$$MASTER
. S'en suit ensuite les différents paramètres.

$$MASTER

title: Titre de la fenêtre

transient: 0 ; ou 1 : enlève la possibilité de mettre en - la fenêtre

column: 0+1 ; Indique que lors du redimensionnement de la fenêtre, la colonne 0 bouge de 1 unité

row: 0+1 ; idem qu'avec column, mais pour la ligne

Les "widgets" possible : $

Vous pouvez utiliser plusieurs widgets comme par exemple une ligne de texte, une étiquette, un bouton, ... Pour cela, utilisez un simple

$
.

Par exemple, pour créer une ligne de texte, procédez comme suit

$ENTRY-Nom-de-ma-zone-de-texte

row:0 ; indique sur quelle ligne on veut placer la zone de texte

col: 0 ; indique sur quelle colonne on veut placer la zone

text: Bonjour ; Met un texte par défaut dans la zone de texte

sticky: we ; fait coller les bords de la zone à l'Est et à l'Ouest (West en anglais)

Les commandes simple

Il est possible de programmer des commandes. Ces dernières peuvent être utilisées par des boutons par exemple. Pour cela, il est important de comprendre le principe de la mémoire.

En revanche, les commandes programmes ne sont pas utilisable directement dans les modules graphiques ! Si vous les placez, une exception sera levé sur l'inexistance de la fonction demandée.

Les commandes sur les variables

La mémoire se compose de variables, que vous pouvez modifier et d'un "buffer", c'est-à-dire, d'une mémoire tampon. Lorsque vous utiliser certaines commandes, elles enregistre dans le buffer le résultat. Vous pouvez ensuite passer le contenu du buffer vers une autre variable.

Pour exécuter une fonction, vous devez utiliser

/nom-de-la-fonction
. Pour en définir une, vous pouvez utiliser la syntaxe suivante
*nom-de-ma-fonction
. Cependant, les options étant limités, les UserForm vous proposent des commandes par défaut pour manipuler la base

/STORE
qui permet d'enregistrer le contenu du buffer vers une variable. Pour l'utiliser, séparer les paramètres par des
+
:
/STORE+nom-de-ma-variable

/PU+Un-texte
Place "Un-texte" dans le buffer

/DEL+variable
Qui permet de supprimer la variable "variable" et son contenu

Les commandes sur les "widgets"

/SET+widget+var
remplace le texte du widget "widget" par le contenu de la variable "var"

/INSERT+widget+var
commande semblable à la précédente, sauf qu'elle ajoute le texte à la suite de celui actuel. Elle n'écrase rien

/GET+widget
met le contenu du texte du widget "widget" dans le buffer

Les commandes système

/OPENFILE+format
ouvre une boîte de dialogue pour choisir un fichier du format "nom-format" et d'extension "extension". Les extensions sont souvent de la forme *.pdf par exemple. Pour d'autres informations sur les formats de fichiers, reportez vous à la section suivante.

/SAVEFILE+format
commande identique la précédente, mais qui propose d'enregistrer un fichier et non d'en ouvrir un. Voir la section ci dessous sur les formats de fichiers.

/SAVE
qui enregistre tous les paramètres de la fenêtre. Ainsi à la fermeture, vous pourrez récupérer le contenu des variables et zones de texte

/EXIT
, commande qui ferme la fenêtre et retourne les variables enregistrées avec la commande précédente.

Les commandes de programmes

Attention, dans les boîtes de dialogue UserForm, les commandes suivantes sont indisponibles. Vous ne pouvez les utiliser que dans les modules normaux.

/ArrangerDuo

/RognerFile

/MarginFile

/ScaleFile

/ExtractPages

/ExtractImages

/SplitFile

Les format de fichiers

Il est possible d'utiliser les différents formats suivants :

-

PDF
Pour ouvrir un fichier PDF

-

SEA
Pour ouvrir une fonction

-

IMG
Pour ouvrir une image

-

CSV
Pour ouvrir une liste de fichiers au format CSV ou Excel

Les commandes **

Il est possible d'utiliser des commandes qui s'execute dès la création de la fenêtre. Ce sont des commandes d'initialisation.

Pour cela, utilisez la syntaxe suivante

; Au lieu de choisir *ma-commande, utilisez **ma-commande :

**initialisation

/PUT+Bonjour

/STORE+text

/SET+LABEL-1+text

/DEL+text
