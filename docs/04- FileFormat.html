Les formats de fichiers

Ce logiciel propose l'enregistrement de données sur plusieurs formats. Il peut aussi en prendre en charge d'autres.

Les fichiers pouvant être lus

PDF

Les PDF (Portable Document Format). C'est le principal format traité par ce logiciel. Il est possible de les lire (dans l'onglet Pages de la fenêtre principale), et de les modifiers. C'est le but principal du logiciel. Pour cela, vous disposez de nombreuses fonctions.

Il est cependant impossible d'éditer un fichier pdf existant depuis ce logiciel. Une fois créé, il est difficile de le modifier.

Cependant, à partir de la seconde version du programme, il est possible de créer des mise en pages avec des variables. Pour plus d'informations, reportez vous à la section Mise en page.

Images

Il est aussi possible de traiter des images : parfois avez vous une image provenant d'un scan, ou d'une capture d'écran (nous vous déconseillons très fortement les captures d'écran pour ne pas perdre en qualité).

Il est ainsi possible de les importer. Lors de l'importation, l'image est converti automatiquement dans un fichier PDF, de telle sorte à occuper le plus d'espace disponible. Les images JPG et PNG sont bien entendu prises en charge. Nous n'avons pas eu l'occasion de tester d'autres formats, tels .GIF, .TIFF, .BMP, et caetera.

Tableaux

Il est possible de lire deux autres formats de fichiers, dans la catégorie tableaux : les fichiers CSV et les fichiers Excel.

Il peut vous arriver d'avoir une liste de fichier à fusionner, et c'est pour cela que nous vous proposons un système de lecture de ces fichiers. Les enregistrements doivent tout de même présenter les règles suivantes :

Tous les enregistrement sont à la suite les uns des autres. Il est impossible de mettre une ligne vide pour séparer ! (sauf à partir de la version 2).

Les fichiers excel doivent être impérativement au format .xlsx, les .xls étant obsolète ne sont pas pris en charge !

Vous devez avoir au moins une colonne avec le nom du fichier. Néanmoins, vous pouvez avoir une autre colonne contenant les pages à conserver et l'inversion du fichier si besoin (OUI ou NON sont les deux seules valeurs acceptées dans cette colonne).

Vous pouvez aussi mettre une ou plusieurs lignes d'entête, car le système d'importation permet de choisir la première ligne des enregistrements.

Note : Pour le format CSV : vous pouvez utiliser n'importe quel séparateur de lignes et de colonnes, il vous suffit de le choisir dans l'interface ! De plus, tous les formats textes sont disponible pour CSV : .csv, .txt, .log, ...

Les fichiers de fonctions

Les fonctions sont stoquées dans des fichiers .pdfsea spéciaux au programme. Vous pouvez en importer, et en modifier depuis les interfaces de choix des programmes.

Lors de la supression d'une fonction, le fichier n'est jamais supprimé par mesure de sécurité, il demeure dans l'arborescence. Cependant, la ligne est suprimée de la base de données des fonctions.

C'est dans cette base de donnée que se trouve la liste des fonctions et leur programme associé, ainsi que si la fonction est en favori ou par défaut. Nous vous déconseillons de changer des paramètres dans cette base de donnée, dans la mesure ou vous pouvez authoriser la modification de fonctions par défaut dans le panneau de configuration.

Ce format est un ZIP dans lequel se trouve

- conditions.ini Contenant l'ensemble des conditions et des équations

- python.py Contenant la programme Python de la fonction (pas encore implémenté avant la version 3)

Les projet

Il est enfin possible d'enregistrer les projets. Vous pouvez inclure dedans une liste de fichiers, des programmes, des modules graphiques, ...

Le fichier est un ZIP, contenant :

- userforms \ Dossier des modules graphiques

- userform1.uf Les modules graphiques sont au format .uf

- userform2.uf

...

- programs \ Dossier des programmes

- program1.prog Les programmes sont enregistrés au format .prog

- program2.prog

...

- liste.csv Liste des fichiers

Vous pouvez lire la section sur les programmes et sur les userforms pour plus d'informations.

IMPORTANT : Tous les fichiers contenant du texte sont encodés en UTF-8 !
